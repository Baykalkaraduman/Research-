# Random Number Generation (`rand()`)

---

# Introduction

Many programs require random numbers.

Examples include

- Games
- Simulations
- Monte Carlo methods
- Cryptography (although `rand()` is **not** suitable)
- Scientific computing
- Statistical sampling
- Random testing
- Artificial intelligence

C++ provides the function

```cpp
rand()
```

through the Standard Library.

To use it,

```cpp
#include <cstdlib>
```

must be included.

---

# Pseudo-Random Numbers

Although we usually call them "random numbers",

the numbers generated by

```cpp
rand()
```

are actually

**pseudo-random numbers**.

This means

- they appear random,
- but they are generated using a mathematical algorithm,
- therefore they are completely deterministic.

If the algorithm starts from the same initial state,

it always produces exactly the same sequence.

---

# rand()

Function

```cpp
rand()
```

Returns

A pseudo-random integer.

Example

```cpp
cout << rand();
```

Possible output

```
1804289383
```

Each call produces another value.

---

# RAND_MAX

The largest possible value returned by

```cpp
rand()
```

is stored inside

```cpp
RAND_MAX
```

Example

```cpp
cout << RAND_MAX;
```

Possible output

```
32767
```

or

```
2147483647
```

depending on the compiler.

---

# Why Do We Need srand()?

Suppose we run this program twice.

```cpp
cout << rand();
cout << rand();
cout << rand();
```

Output

Run 1

```
41
18467
6334
```

Run 2

```
41
18467
6334
```

Exactly the same.

Why?

Because the random generator always starts from the same seed.

---

# Seeding the Generator

To generate a different sequence,

use

```cpp
srand()
```

Example

```cpp
srand(time(nullptr));
```

This initializes the generator using the current system time.

Since the time changes every second,

the sequence also changes.

Required header

```cpp
#include <ctime>
```

---

# Why time(nullptr)?

Function

```cpp
time(nullptr)
```

returns the current calendar time.

Example

```
1719345837
```

Every second,

this value changes,

making the seed different every program execution.

# Generating Numbers Within a Range

Suppose we need numbers between

```
1 and 6
```

Instead of

```cpp
rand()
```

we write

```cpp
rand() % 6 + 1
```

Possible outputs

```
1

2

3

4

5

6
```

---

# General Formula

The safest formula is

```cpp
rand() % (max - min + 1) + min
```

Example

Numbers between

```
20

50
```

```cpp
rand() % (50 - 20 + 1) + 20
```

Possible outputs

```
20

21

...

50
```

To obtain a number between


# Probability Distribution

The values generated by

```cpp
rand()
```

are intended to be approximately uniformly distributed.

For example,

rolling a die

100000 times

should produce roughly

```
1 → 16.7%

2 → 16.7%

...

6 → 16.7%
```

Although the values are not perfectly uniform,

they become closer as the number of samples increases.

---

# Limitations of rand()

Although useful for learning,

`rand()` has several disadvantages.

- Low-quality randomness
- Small period on some systems
- Compiler-dependent implementation
- Not suitable for cryptography
- Not suitable for high-quality simulations

Because of these limitations,

Modern C++ recommends using

```cpp
<random>
```

instead.

---

# Modern C++ Random Library

Since C++11,

the preferred approach is

```cpp
#include <random>
```

Example

```cpp
std::mt19937
```

and

```cpp
std::uniform_int_distribution
```

These generators

- produce higher quality random numbers,
- are reproducible,
- support many statistical distributions.

We will study this library later.

# Best Practices

✔ Seed the generator only once.

```cpp
srand(time(nullptr));
```

✔ Use the general range formula.

```cpp
rand()%(max-min+1)+min
```

✔ Prefer `<random>` in Modern C++.

✔ Never use `rand()` for security applications.

✔ Use descriptive variable names.

---

# Common Mistakes

## Forgetting to seed

Incorrect

```cpp
rand();
```

Every execution produces the same sequence.

---

## Seeding multiple times

Incorrect

```cpp
for(...)

    srand(time(nullptr));
```

The generator should usually be seeded only once.

---

## Incorrect range formula

Incorrect

```cpp
rand()%max+min
```

This does **not** work correctly unless

```
min==1
```

Prefer

```cpp
rand()%(max-min+1)+min
```

---

## Using rand() for passwords

Never use

```cpp
rand()
```

for

- passwords
- encryption
- authentication

Use cryptographically secure generators instead.
